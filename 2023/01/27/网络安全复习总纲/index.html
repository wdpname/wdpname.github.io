1.1 网络安全攻防博宇的套路

攻防目标

  • 联网的实体设备
    • 传统的联网设备（蓝字）
      • 计算机，计算机网络，服务器，交换设备
    • 物联网设备
      • 手机，摄像头，各式传感器
      • 应用场景：工控，智慧城市，车联网，智慧家居等
  • 人（社会工程学）
    • 个体
    • 企业

攻击目的

  • 设备：控制权 + 数据
    • 控制权：断网，断电，断水
    • 数据：身份数据，金融数据，用户数据
  • 人：骗财骗色

攻防博弈

  • 攻：

    • 收集数据，从而发现目标手机号，账号，位置，邮箱，单位，资产

    • 借助手段，实施攻击

      • 夺取控制权
        • 后门
        • 恶意代码
        • 漏洞利用
        • DDos攻击
      • 窃取数据
        • 恶意代码
        • 漏洞利用
        • 网络嗅探
        • 口令爆破
        • 中间人攻击
        • 网络钓鱼

    • 隐藏踪迹，清除痕迹

      • 隐藏踪迹

        • 本地隐藏技术

          • 文件隐藏：文件名更名为合法文件名
          • 进程隐藏：附着系统进程，以合法身份运行
          • 网络连接隐藏：复用当前网络服务端口，如80
          • 编译器隐藏：攻击者是编译器开发人员，实施原始分发攻击

        • 网络隐藏技术

          • 通信内容隐藏：对通信内容加密
          • 传输信道隐藏：一个进程能够直接或者间接访问某存储空间，而该存储空间又能被另外一个进程访问，构成隐蔽存储通道

      • 清除痕迹

        • 清除日志和操作记录
          • 如windows事件日志清除
          • 清除IIS日志

  • 防：

    • 事前 防护（依靠技术）
      • 加密，脱敏，防火墙，身份认证访问控制
    • 事中 检测 （分析行为）
      • 精准检测：通过系统环境数据和应用进程数据，网络/主机流量来捕获信息，然后通过结合相关技术，来智能分析实现检测行为。
    • 事后 恢复，取证，溯源，反制
      • 取证：利用蜜罐获取攻击指纹
      • 溯源：描绘攻击者图像
      • 反制：植入脚本或探针实现绝地反制

网络空间攻击链 （Cyber Kill Chain）

侦查 武器构建 载荷投递 漏洞利用 安装植入 命令控制 目标达成

攻击时间轴

漏洞分类

恶意代码分类

  • 计算机病毒：可以自我复制和感染其他计算机的恶意代码。
  • 木马：可以伪装成他类的程序。
  • 蠕虫：像病毒那样可以扩散，但蠕虫可以自我复制，不需要借助其他宿主。
  • Rootkit：是攻击者用来隐藏自己踪迹和保留root访问权限的工具。
  • 勒索病毒：利用各种加密算法对文件进行加密，被感染者一般无法解密，必须拿到解密的私钥才有可能破解。
  • 逻辑炸弹：可以有某类事件触发执行。例如一个时间炸弹（再某一特定时刻触发执行）。
  • 僵尸网络：是由C&C服务器以及僵尸牧人控制的僵尸网络。
  • 间谍软件：间谍软件就是能偷偷安装在受害者电脑上并收集受害者的敏感信息的软件。
  • 广告软件：自动生成广告的软件。
  • 移动恶意代码：在移动设备上传播。

3.3 网络动态防御技术

通过主动的重构系统来阻止或降低网络攻击的影响

基本思想：通过动态地改变，伪装目标网络系统的特征，以增加攻击代价，提高目标弹性，增强防御能力。

移动目标防御

MTD（Moving Target Defense）

  • 五个层次
    • 动态网络：指网络拓扑，网络配置，网络资源，网络节点，网络业务等网络要素动态化，虚拟化，随机化。
    • 动态平台：构建多样化的运行平台，动态改变应用运行的环境
    • 动态运行环境：操作系统提供给应用程序地执行环境动态改变
    • 动态软件：应用程序代码动态变化
    • 动态数据：根据系统的防御需求，动态化更改相关数据的格式，句法，编码或者表现形式。

网络空间拟态防御

CMD Cyberspace Mimic Defense

在不改变网络功能本质的前提下，动态改变其攻击表面。

应用

  • 数据校验
  • 拟态交换机
  • Web服务器模型
  • 拟态蜜罐

4.1 入侵检测+防火墙

入侵检测

  • 入侵（Instrusion）
    • 非法取得系统控制权
    • 利用系统漏洞收集信息
    • 破坏信息系统
  • 入侵检测（Instrusion Detection）
    • 检测对系统的非授权访问
    • 监控系统运行状态，保证系统资源的保密性，完整性，可用性
    • 识别针对计算机系统网络系统或广义信息系统的非法攻击

入侵检测系统（IDS）的分类

  1. 按照数据来源分类：

    • 基于网络的IDS （NIDS）

    • 基于主机的IDS （HIDS）

    • 分布式IDS （DIDS）

  2. 按照检测策略分类：

    • 误用检测
    • 异常检测
    • 完整性分析

防火墙

防火墙是由软件和硬件组成的系统，它处于安全的网络和不安全的网络之间，属于边界防护设备，由系统管理员设置访问控制规则，对进出网络边界的数据流进行过滤。

分类

  1. 逻辑上讲：
    • 主机防火墙
    • 网络防火墙
  2. 物理上讲：
    • 硬件防火墙
    • 软件防火墙

发展史

包过滤 —> 应用代理 —> 状态检测 —> UTM（Unified Threat Management） —-> NGFW（Next Generation Firewall）

UTM：统一威胁管理 （Unified Threat Management）

NGFW：下一代防火墙 （Next Generation Firewall） 不在使用基于端口，而是通过应用程序来执行相关的安全策略

WAF

Web Application Firewall

Web应用程序防火墙过滤，监视和阻止与web应用程序之间的HTTP流量

WAF与常规防火墙的区别在于，WAF能够过滤特定web应用程序的内容，而常规防火墙则充当服务器之间的安全门

防火墙和入侵检测的区别

IDS vs IPS vs FW vs WAF

IDS：入侵检测技术

IPS：入侵防御技术

FW：防火墙

WAF：web应用程序防火墙

4.3 恶意代码

定义：在明确提示用户或未经用户许可的情况下，在用户计算机或其他终端上安装运行，侵犯用户合法权益。

基本特征

  • 一类完整程序，也是由人编制的，而不是在计算机环境或系统中自生的
  • 对系统具有破坏性或威胁性，非授权性
  • 具有潜伏性，传染性，依附性

生命周期

编制代码 —> 传播 —> 感染 —> 触发 —> 运行

  1. 传播机制
    1. 文件流动
    2. 网页脚本和插件
    3. 电子邮件
    4. 数字内容播放
    5. 网络攻击
    6. 自我传播
    7. 扫二维码
  2. 感染机制
    1. 感染引导系统
    2. 感染执行文件
    3. 感染结构化文档
    4. 感染网络服务或客户端
    5. 假冒文件

4.4 APT攻击

APT （Advanced Persistent Threat）高级持续性威胁

是一种以商业或者政治目的为前提的特定攻击，其通过一系列具有针对性的攻击行为以获取某个组织甚至国家的重要信息

常常采用多种攻击技术手段，包括一些最为先进的手段和社会工程学方法，并通过长时间，持续性的网络渗透，一步步的获取内部网络权限，此后便长期潜伏在内部网络，不断地收集各种信息，直至窃取到重要情报。

特征

  • 高级
  • 持续性
  • 隐蔽性

四个要素

  • 攻击者
  • 攻击目的
  • 攻击手段
  • 攻击过程

5.1 攻击取证，溯源，反制

检测 -> 响应 -> 取证 -> 溯源 -> 反制

  • 检测：收集和分析网络行为，安全日志，审计数据，其他网络上可以获得的信息以及计算机系统中若干关键点的信息，检查网络或系统中是否存在违犯安全策略的行为和被攻击的迹象。
  • 响应：在遇到突发事件后锁所采取的措施和行动
  • 取证：结合流量，web访问日志，终端系统或者软件日志等信息来挖掘或者推断相关线索
  • 溯源：将碎片化安全事件线还原成完整的攻击链，全面还原攻击事件及其背景，快速定位攻击源头
  • 反制：以其人之道换其人之身

5.2 软件供应链安全

软件供应链定义：指在软件开发中所涉及到的组件，开发流程以及投入生产和最终软件产品分发的过程

软件供应链组成：

  • 硬件和基础架构
  • 操作系统
  • 编译器和编辑器
  • 驱动和依赖项
  • 开源脚本和打包好的软件
  • 代码仓库引擎，测试套件以及CI/CD工具
  • 云服务和数据中心

软件供应链安全：将风险管理和网络安全原则相结合，检测第三方组件给软件带来的风险，并将其最小

软件开发生命周期安全风险治理

需求分析阶段

  • 安全需求分析
  • 安全设计原则
  • 确定安全标准
  • 攻击面分析
  • 威胁建模
  • 安全隐私需求设置知识库

开发测试阶段

  • 安全编码
  • 管理开源及第三方组件安全风险
  • 变更管理
  • 代码安全审查

  • 开源及第三方组件确认

  • 配置审计

  • 安全隐私测试
  • 漏洞扫描
  • 模糊测试
  • 渗透测试

发布运营阶段

  • 发布阶段：确保可以交付安全的软件产品
  • 运营阶段：保障软件产品可以稳定运行

停用下线阶段

软件停用下线前需提供满足客户使用的更新软件，确保客户使用过程衔接。

软件停用下线后需保护云服务用户的隐私与数据安全。

软件开发模型

  • 瀑布模型
  • DevOps

DevSecOps

指通过与IT安全团队，软件开发人员和运营团队合作，在标准DevOps周期中建立关键的安全原则

安全开发生命周期（SDL）

DevSecOps工具

软件组件分析（SCA）

是一种用于管理开源组件应用安全的方法

通过SCA，开发团队可以快速跟踪和分析项目的开源组件

静态应用安全检测工具（SAST）

通常在编码阶段分析应用程序的源代码或者二进制文件的语法，结构，过程，接口等来发现程序代码存在的安全漏洞

动态应用安全检测工具（DAST）

在测试或运行阶段分析应用程序的动态运行状态，它模拟黑客行为对应程序进行动态攻击，分析应用程序的反应，从而确定该web应用是否易受攻击。

交互式应用安全检测工具（IAST）

通过在服务端部署Agent程序，收集，监控Web应用程序运行时函数执行，数据传输，并与扫描器端进行实时交互，搞笑，准确的识别安全缺陷及漏洞，同时可准确确定漏洞所在的代码文件，行数，函数及参数。
