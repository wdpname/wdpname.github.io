A review of research on malicious code visual detection technology

发表于2023-10-27|更新于2023-11-25
|阅读量:

1 传统恶意代码检测方法

1.1　静态检测技术

  • 优点

    • 能耗小、风险低、速度快，且对实时性要求低
    • 对恶意样本覆盖率高，可快速捕获语法和语义信息进行全面分析
    • 对分析系统不会造成破坏，较为安全
    • 可以在执行文件前对代 码流程有一定掌握，此种方法不会受到具体 进程执行流程的制约，可以对代码进行详尽的细 粒度分析
    • 静态分 析速度快、效率高、适合大规模实时分析

  • 缺点

    • 受到经变形、多态、代码混淆技术处理后的恶意代码干扰，会出现误报和漏报的情况。其问题在于难以应对逐渐复杂的恶 意样本加壳与混淆技术，

  • 静态特征

    • 字节序列特征 —> N-Gram方法提取
    • PE文件 也可作为特征
    • API序列
    • 可阅读字符串
    • 熵
    • 动态链接库（DLL）

  • N-Gram方法

    • 缺点

      方法单一、效率较低

虽然有这些静态特征可以使用，但随着对抗反汇编、反调试、反虚拟机、加壳与脱壳等恶意代码反检测技术的发展，使得传统静态检测方法提取的特征不能准确代表恶意代码攻击信息

1.2　动态检测技术

  • 优点
    • 能识别新型恶意代码
    • 有效处理静态检测中存在的误报、漏报等问题
    • 在一定程度 上解决了软件加壳与混淆的问题
  • 缺点
    • 恶意代码对运行环境要求高
    • 所需的时间成本高
    • 面临一些安全问题，比如内核级的特殊权限、网络连接等
    • 后动态分析方法提取特征 复杂、检测时间较长、占用系统资源较多和误报 率较高。
    • 对资源消耗巨大，在应用到大规模数据 时，这种方式的弊端尤为凸显。

1.3　混合检测技术

对于加壳的恶意代码，就适用于混合检测技术。

基本思路：先通过动态分析对加壳代码进行脱壳，然后通过静态分析脱壳代码。

  • 优点
    • 涵盖了静态与动态检测的优点
  • 缺点
    • 消耗大量资源
    • 混合检测的复杂度与工作量使这种方式不适用于大规模数据集

3 恶意代码可视化检测技术研究

3.1　基于机器学习的恶意代码检测

  • 原理
    • 将恶意代码可视化为图像后，采用基于机器学习的恶意代码检测方法进行检测，其原理是通过图像处理技术从图像中提取不同类型特征，描述待分析样本的恶意攻击行为，每个恶意代码样本均由一个降维后的固定维度向量表示，然后使用机器学习算法对已知标签样本图像进行训练，构建分类器，训练出分类器后便可对未知样本进行检测分析。

  • 难点

    • 在于如何使用特征分析技术提取出不易受随机因子干扰且具有家族代表性的信息

  • 常见的特征分析技术

    • 特征抽象
      • 将源数据抽象成算法可理解的数据
    • 特征降维
      • 特征选择
        • 从原有特征集中选出子集，仅单纯进行挑选不改变特征性质
      • 特征提取
        • 在原有特征集基础上创造一个新的特征子集，通过空间变换改变了特征性质

3.2　基于深度学习的恶意代码检测

  • 优点
    • 能自动学习、提取特征
    • 减少人工提取特征的时间成本
    • 速度快
    • 资源消耗少
  • 缺点
    • 不能深层次提取特征
    • 模型结构复杂
    • 参数过多
    • 模型泛化能力不足

4 面临的问题及分析

  • 存在的问题
    • 恶意代码形态多样化
    • 标准数据集较少
    • 数据分布不均衡
      • 若正负数据样本数量或不同家族的样本数量差距较大，将不利于深度模型学习样本内在特征
      • 只有输入足够且均衡的训练数据才能有效避免过拟合现象，如果数据集较小，可通过数据增强技术增加样本数量，抑制数据集不平衡带来的不良影响
    • 数据缺乏标签
  • 目前公开的基于windows的恶意软件数据集
    • Malimg
    • Malheur
    • VirusShare
    • Microsoft Kaggle
