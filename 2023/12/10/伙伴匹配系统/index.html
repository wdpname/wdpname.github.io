对于前端请求后端，一般使用axios，一般都要新建一个myAxios.js或者myAxios.ts文件，导出一个我们自定义的一个axios的对象，因为原始的axios对象不好用，我们要设置一个基础的请求地址，一般要提前定义一个ip和端口的baseUrl.

从中可以得出结论：数据库中存储json形式的字符串时，需要将每个内容(如 java)加上双引号””，这样才是正确的json形式。

一般axios请求 形式

const userListRes: UserType[]= await MyAxios.get(‘/user/search/tags’, {
params: {
tagNameList: tags,
},
paramsSerializer: params => { // 使得能够正确的传递 诸如数组形式的 请求参数
return qs.stringify(params, {indices: false})
}
}).then((response) => {
console.log(‘/user/search/tags success’, response);
showSuccessToast(‘请求成功’);
return response.data?.data;
// userList= response.data.data;
})

这里得出结论： 前端传递不是基础类型的参数给后端，需要json序列化，同样：后端将数据给前端，需要在后端或者前端进行反序列化（json解析将字符串转换为非基础类型的对象）。
