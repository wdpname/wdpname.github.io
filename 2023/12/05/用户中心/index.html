4.2 用户注册

4.2.1 注册逻辑

  1. 用户在前端输入账户和密码、以及校验码()

  2. 校验用户的账户、密码、校验密码，是否符合要求

    1. 非空
    2. 账户的话不小于4位
    3. 密码就不小于8位吧
    4. 账户不能重复
    5. 账户不包含特殊字符
    6. 密码和校验密码相同
  3. 对密码进行加密（密码干万不要直接以明文存储到数据库中)
  4. 向数据库插入用户数据

4.2.2 注册实现

在进行校验时，每一个参数都要判空，非常麻烦，可以引入apache common lang包，非常好用！！！

4.3用户登录

4.3.1登录接口

接受参数：用户账户、密码
请求类型：POST
请求体：SQN格式的数据

请求参数很长时不建议用get

返回值：用户信息（脱敏)

4.3.2 登录逻辑

  1. 校验用户登录是否合法
    1. 非空
    2. 账户的话不小于4位
    3. 密码就不小于8位吧
    4. 账户不包含特殊字符
  2. 校验密码是否输入正确，要和数据库中的密文密码去对比
  3. 返回用户信息（脱敏，隐藏敏感信息）：脱敏其实就是在从数据库获取到一个对象时，新生成一个对象，赋值想要给他的值
  4. 我们要记录用户的登录态(session),将其存到服务器上
    (用后端SpringBoot框架封装的服务器tomcat去记录)

4.3.3如何知道那个用户登录了

  1. 连接服务器端后，得到一个session1状态，返回给前端
  2. 登录成功后，得到了登录成功的session,并且给该sessio
    设置一些值（比如用户信息），返回给前端一个设置
    cookie的”命令”
  3. 前端接收到后端的命令后，设置cookie,保存到浏览器内
  4. 前端再次请求后端的时候（相同的域名），在请求头中带
    上cookie去请求
  5. 后端拿到前端传来的cookie,找到对应的session
  6. 后端从session中可以取出基于该session存储的变量（用
    户的登录信息、登录名)

在测试接口时，填写post请求体时，只包含数字的字段，比如password string类型的也可以用数字编写，json会自动转换为字符串。
