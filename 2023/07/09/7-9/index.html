leetcode

  • 问题1：给定一个数组，你可以选择那一天买入股票，之后选择那一天卖出股票。使得在这一个数组这些天过去之后能够开始使得利润最大。

    找到若干个不重叠的区间，每个区间（i，j] 为什么是左开右闭呢？左开有闭保证了区间的不重叠。在一个区间对应的元素相减等价于在这个区间的每个相邻的元素相减的值的非负数。每个区间的相减值等价于 这个区间的每个相邻元素的相减（非负数），而每个不重叠的区间相叠加就形成了一个完整的数组区间，所以，我们只需要遍历一次数组，每次将相邻的元素取差值 与 0 取最大值，结果叠加即可。

  • java方法是存储在JVM的那个区

    个人理解：方法是存储在方法区的，准确来说，方法的指令代码是存储在方法区中的。而我们常说的在运行一个方法的时候会在栈区中开辟一块内存，这里的内存其实存储的方法的形参和方法内声明的局部变量（本地变量）。我的理解是，对于new出来的对象是存储在堆区中的，而通过new出来的对象调用实例方法时，其实是先通过堆中的new对象从对象头中找到这个对象的类在方法区中的地址，然后找到这个对象的类，而方法区中的这个对象的类中保存了一个方法表，表中有这个对象调用的方法的代码。

    基本上我们打交道的就是 方法区 栈区 堆区

  • 对于 A，B两堆url链接，AB都有很多的url链接，并且内存都存不下，怎么查找出A，B公共的url链接？

    想到的最简单的方法就是，写两个for循环，依次遍历来暴力查找其中公共的url链接。而这是不行的，非常的shabi

    而再仔细想想，遍历不行，可以用二分啊，但是又有一个问题，我们不能一次性在内存中存下这么多的数据，所以必须将A,B

    这些数据集分成许多个小文件，按照什么规则进行分类呢，一开始想到的往往总是最简单最耗时的算法，如按照url长度分类，可以在巨大的数据样本下，有可能分出的小文件也有可能在内存中存不下，这是完全有可能的。显然就不能使用按照url长度分类，这时我们就应该想到使用hash分类，比如将url链接经过md5输出的首个字母分类，这样就能分类成256中小文件（md5输出的可能每个字母为0123456789abcdef）,如果还是不行，那么可以按照前两个字母分类。这样总能满足小文件能够存在内存中。这样就可以操作了。

    模板：l = 0， r = n-1; while(l <= r); c = l + (r - l) / 2; l = c+1, r = c-1;
