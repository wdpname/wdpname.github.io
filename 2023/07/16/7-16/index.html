  2. lambda表达式 == 匿名类实现

    1. 识别 你要编写的lambda表达式的类型

    2. lambda 不能捕获变量，只能捕获值，所以在lambda方法体内不能使用方法体外的变量。

      也就是说，在lambda方法体内使用外部变量，编译器会自动的将这个变量声明为final变量，如果在之后的代码中，有将这个变量修改，那么就会报错：

      现在有一个疑问：为啥lambda表达式中必须将变量声明为final类型

      我是这么理解的，lambda表达式的方法体只有访问局部变量且是lambda表达式外的时候，才会有这个编译器自动的转变类型，要是使用的是实例变量，那么就不会有这个问题了。可以大胆的使用lambda表达式了，不会报错。这个就牵扯到了jvm虚拟机的内存分配了。因为实例变量是放在堆上的，供所有线程共享，而局部变量是存储在栈上的，lambda表达式比如实现Runnable接口，而这个接口是关于多线程并发的，而这个时候lambda实现runnable接口开启多线程来捕获局部变量，可能有并发的问题。归根到底一句话，想要确保lambda表达式访问的局部变量是最新的值。

      参考文档：https://segmentfault.com/a/1190000022757341 为什么 lambda表达式捕获的局部变量必须是final 或者 effectivly final呢？

  3. 函数式接口

    想要使用 lambda表达式，那么必须要是函数式接口，什么是函数式接口，其实就是接口中只有一个抽象方法。

  4. 什么是Stream API

  1. ubuntu 虚拟机安装 之 分区手动配置

    swap区：设为逻辑分区、作用跟电脑内存类似，大小也可以和电脑内存一样大；
    /boot区：设为主分区，引导系统的，分配200M-500M就可以，条件好些分配1G，不过一般200M就可以；
    / 区(主分区)：设为主分区，存放系统，类似windows系统的C盘，大小根据电脑性能分配，一般建议存储空间的20%-35%；
    /home区：设为逻辑分区，储存用户数据文档的，剩下的存储空间全部给它就可以。
