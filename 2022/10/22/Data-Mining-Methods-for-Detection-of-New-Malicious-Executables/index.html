论文发布时间：2000年

该数据挖掘方法 VS 基于签名的传统方法

优点

  1. 检测检测率1倍之多

  2. 自动化生成

该数据挖掘算法的数据是怎么处理的

所有的信息在不执行未知程序的情况下获得的，检查二进制文件的静态属性获得的。

提取的三种特征

  1. 二进制文件使用DLL的列表
  2. 二进制文件的DLL函数调用列表
  3. 每个DLL的不同函数调用的数量

二进制分析的第一种方法：

1. 使用了二进制加载的DLL作为特征，特征向量由30个布尔值组成，表示二进制文件是否使用DLL

二进制分析的第二种方法：

  1. 使用DLL及其函数调用作为特性。与第一种方法类型，特征向量由2229个布尔值组成的。

第三种方法：

  1. 统计了每个DL中使用的不同函数调用的数量，特征向量包括30个整数值。

但是由于不能对整个数据集进行分析，所以使用了另外一种方法。

GUN字符串程序

​ 由于PE可执行文件和非PE可执行文件都有字符串编码，我们可以使用GNU字符串程序提取特征。找到我们数据集中的一些常见字符串如下。

算法

基于签名的方法

检测恶意软件样本的中找到的字节序列，将这些字节序列连接在一起，为每个恶意软件生成唯一的签名。

RIPPER

  1. 是一个归纳规则学习器
  2. 设定四个特征如下：
    1. 它有没有GUI
    2. 是否执行恶意功能
    3. 是否危及系统安全
    4. 是否会删除文件

Naive Bayes

  1. 使用数据：字符串和字节序列都用到

  2. 理论过程

$P(C \mid F)=\frac{P(F \mid C) P(C)}{P(F)}$ ——》 $P(C \mid F)=\frac{\prod{i=1}^{n} P\left(F{i} \mid C\right) P(C)}{\prod{j=1}^{n} P\left(F{j}\right)}$ —> $\text { Most Likely Class }=\max {C}\left(P(C) \prod{i=1}^{n} P\left(F_{i} \mid C\right)\right)$

C 是表示是恶意类别还是干净类别，F表示样本的特征

$F_i$ 代表样本的第i个特征，

最后结果返回最高概率对应的C类别

  1. 实验过程

    为了训练分类器，我们记录了每个类中包含每个独特特征的程序数量。我们使用这些信息将新程序分类为适当的类。我们首先使用特征提取来确定程序中包含的特征。然后我们应用等式 $\text { Most Likely Class }=\max {C}\left(P(C) \prod{i=1}^{n} P\left(F_{i} \mid C\right)\right)$ 来计算程序的最可能类。

  2. 结论：使用了朴素贝叶斯算法，计算了字节序列和字符串的最可能类别

  1. 结构：其实就是多个朴素贝叶斯分类器对最终结果进行投票
  2. 实践过程：将数据集分为N组，每组训练一个朴素贝叶斯分类器，

Naive Bayes

一个字符串出现在某个类的概率 == 该字符串出现在该类的训练集中的总次数/该字符串在整个训练集出现的总次数

在上例中，字符串“windows”更可能出现在良性程序中，而字符串“*.COM”更可能发生在恶意可执行程序中。

在这种预测算法中，有一点缺陷，如：如果一个字符串“hello，world”仅出现在良性软件集合中，那么这个预测算法会将以后每个文件中含有这个“hello，world”字符串都将视为良性软件，那么就起不到预测的效果。

所以在上图中，字符串“.COM”不会出现在任何良性程序中，因此良性类中出现“.COM“的概率近似为1/12，而不是0/11。这是一种优化方案。

在这个实验中，我们更加关心检测率：$\frac{T P}{T P+F N}$ ，和 错误率：$\frac{F P}{T N+F P}$ ，准确率：$\frac{T P+T N}{T P+T N+F P+F N}$

总结：这些是根据算法和功能对新恶意程序进行分类的结果。使用字节的多朴素贝叶斯具有最高的检测率，使用字符串的签名方法具有最低的误报率。总体准确性最高的是使用字符串的朴素贝叶斯算法。注意，基于签名的方法的检测率低于数据挖掘方法。

这篇论文创新的地方

该论文第一个贡献是一种检测以前无法检测到的恶意可执行文件的方法。我们通过将我们的结果与传统的基于签名的方法和其他学习算法进行比较来证明这一点。多朴素贝叶斯方法在未知程序上具有最高的准确率和检测率，为97.76%，是基于签名的方法检测率的两倍以上。方法将基于签名的方法对新恶意二进制文件的检测率提高了一倍多。本文中讨论的方法被实现为网络邮件过滤器，阻止恶意可执行文件在网络级别上复制将是非常有利的。

不足之处

实验数据较小，应当在更大的数据集上跑模型
